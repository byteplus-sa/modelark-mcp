# Security Policy

## Supported versions

Security fixes are applied to the latest release in the `0.1.x` series and to
the current `main` branch. Earlier versions are not supported.

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in public issues, pull requests, or
discussions.

While this repository remains private, report concerns through an established
private BytePlus/ByteDance maintainer channel. After the repository is public,
use GitHub's **Report a vulnerability** flow, which will be enabled before the
visibility change.

Include enough information to reproduce the issue, its potential impact, and
any suggested mitigation. We will acknowledge reports promptly, coordinate a
fix privately, and publish an advisory only after a fix is available.

## Secrets

Never include provider API keys, access tokens, generated credentials, or
customer data in a report. If a credential is exposed, revoke or rotate it
immediately and report the exposure through a private maintainer channel.
